Подробное описание

Значение ключа - произвольный json-объект, строка, число или флаг. Объект сохраняется в базу как есть, без десериализации, в таком же виде и возвращается при запросе.

Данные класса

◆ prefix

std::string wamba::prefixdb::request::set::prefix

Обязательный параметр. Если БД префикса не существует, то она создается автоматически.

Возвращаемые значения
response::set::statusEmptyPrefix - если не задан префикс или пустая строка
response::set::statusCreatePrefixFail - не смог создать новый префикс
response::set::statusPrefixLengthExceeded - превышен допустимый размер в имени префикса

◆ sync

bool wamba::prefixdb::request::set::sync = false

Устанавливает соответствующий флаг при записи в RockDB, а также игнорирует отложенную запись через очередь, если такая разрешена в конфигурации db_config::enable_delayed_write.

◆ nores

bool wamba::prefixdb::request::set::nores = true

если true (по умолчанию), то пустой результат в ответе. В противном случае происходит вычитывание всех полей, которые были записаны и возврат значения каждого поля (noval==**false**) или статус чтения true|**false** (noval==**true**) Если задан nores==**true**, то noval игнорируется

◆ noval

bool wamba::prefixdb::request::set::noval = false

Действет только если nores==false. Если задан, но вы результате запроса будут записаны вместо значения ключей статус чтения (true или false)

◆ snapshot

size_t wamba::prefixdb::request::set::snapshot = 0
